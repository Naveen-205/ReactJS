### 1Ô∏è‚É£ Why React was introduced?

**Answer:**
React was introduced to make building **dynamic and interactive user interfaces (UI)** easier and faster.
Before React, developers had to manually update the webpage every time data changed ‚Äî which was slow and complex.
React solves this by using a **Virtual DOM** that updates only the changed parts of a webpage efficiently.

**Example:**
When you ‚Äúlike‚Äù a post on Facebook, only that like button updates ‚Äî the entire page doesn‚Äôt reload.
‚û°Ô∏è This is possible because of React‚Äôs **Virtual DOM**.

### 2Ô∏è‚É£ What is React?

**Answer:**
React is a **JavaScript library** used for building **user interfaces (UI)**, especially **single-page applications (SPA)**.

It helps developers create reusable UI components like buttons, forms, cards, etc.

**Example:**

```jsx

  return <h1>Hello, Naveen!</h1>;
}
```

This small code creates a **component** that displays ‚ÄúHello, Naveen!‚Äù

### 4Ô∏è‚É£ Why do we use React?

**Answer:**

### 7Ô∏è‚É£ What is TypeScript?

**Answer:**
TypeScript is a **superset of JavaScript** that adds **type safety**.
It helps catch errors during development (before running the code).

**Example:**

```typescript

age = "twenty";        // ‚ùå Error: Type 'string' is not assignable to type 'number'
```

TypeScript makes code more reliable and easier to debug ‚Äî it‚Äôs commonly used with **React and Angular**.

## üü® **JAVASCRIPT QUESTIONS**

### 8Ô∏è‚É£ What is a Variable?

**Answer:**
A **variable** is used to store data in memory.
Example:

```javascript
let name = "Naveen";
let age = 21;
```

Here, `name` and `age` are variables.

---

### 9Ô∏è‚É£ What is Declaration?

**Answer:**
**Declaration** means creating a variable (without giving value).
Example:

```javascript
let a;  // declaration
```

---

### üîü What is Initialization?

**Answer:**
**Initialization** means assigning a value to a declared variable.
Example:

```javascript
a = 10;  // initialization
```

Or together:

```javascript
let a = 10;  // declaration + initialization
```

---

### 11Ô∏è‚É£ What is Scope?

**Answer:**
**Scope** means the area where a variable can be accessed.
There are mainly:

* Global scope
* Function scope
* Block scope

---

### 12Ô∏è‚É£ What is Lexical Scope?

**Answer:**
**Lexical scope** means **a function can access variables from its parent (outer) function**.

**Example:**

```javascript
function outer() {
  let name = "Naveen";
  function inner() {
    console.log(name); // inner can access outer variable
  }
  inner();
}
outer(); // Output: Naveen
```

---

### 13Ô∏è‚É£ What is Block Scope?

**Answer:**
Variables declared inside `{ }` using `let` or `const` are **block scoped** (cannot be accessed outside).

**Example:**

```javascript
{
  let x = 10;
}
console.log(x); // ‚ùå Error: x is not defined
```

---

### 14Ô∏è‚É£ What is Functional Scope?

**Answer:**
Variables declared inside a function using `var` are **function scoped** (accessible only inside that function).

**Example:**

```javascript
function demo() {
  var x = 20;
}
console.log(x); // ‚ùå Error: x is not defined
```

---

### 15Ô∏è‚É£ What is Hoisting?

**Answer:**
**Hoisting** means JavaScript moves variable and function declarations to the top of their scope before execution.

**Example:**

```javascript
console.log(a); // undefined
var a = 10;
```

Here, JS internally does this:

```javascript
var a;
console.log(a); // undefined
a = 10;
```

---

### 16Ô∏è‚É£ What are Closures?

**Answer:**
A **closure** is a function that remembers the variables from its outer function even after the outer function has finished executing.

**Example:**

```javascript
function outer() {
  let count = 0;
  return function inner() {
    count++;
    console.log(count);
  }
}

const counter = outer();
counter(); // 1
counter(); // 2
```

The `inner` function remembers `count` even after `outer()` is done ‚Äî that‚Äôs a closure.

### 17Ô∏è‚É£ In web development process ‚Äî why Node is used? Which came first: Node, React, or Angular?

**Answer:**

* **Node.js** is a **runtime environment** that lets JavaScript run **outside the browser** (like on a server).
* It allows developers to build **backend (server-side)** applications using JavaScript.

**Example:**
Before Node.js, JavaScript could only run inside browsers.
With Node.js, we can create servers, connect databases, and build APIs.

üïì **Order of release:**

1. **Node.js** ‚Äì 2009
2. **AngularJS** ‚Äì 2010
3. **React.js** ‚Äì 2013

‚û°Ô∏è So, **Node.js came first**, then Angular, then React.
